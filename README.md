# aws-common-questions

AWS services commonly used for building microservices architectures:

## 1. Compute

- **AWS Lambda** — Serverless compute for event‑driven microservices
- **Amazon ECS / EKS** — Container orchestration for microservices in Docker/Kubernetes

## 2. Storage

- **Amazon S3** — Object storage for files, assets, and backups
- **Amazon EFS / EBS** — File/block storage for persistent data

## 3. Messaging & Event Streaming

- **Amazon SQS** — Message queuing for decoupling microservices
- **Amazon SNS** — Pub/Sub messaging for notifications and fan‑out
- **Amazon Kinesis** — Real‑time data streaming and analytics
- **Amazon EventBridge** — Event bus for event‑driven architectures

## 4. API Management

- **Amazon API Gateway** — Expose, manage, and secure RESTful APIs

## 5. Database

- **Amazon DynamoDB** — NoSQL database for scalable, low‑latency data
- **Amazon RDS / Aurora** — Managed relational databases

## 6. Monitoring & Logging

- **Amazon CloudWatch** — Monitoring, logging, and alerting
- **AWS X-Ray** — Distributed tracing for microservices

## 7. Security & Identity

- **AWS IAM** — Access control and permissions
- **AWS Cognito** — User authentication and authorization
